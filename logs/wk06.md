# You found a website you love. How would you figure out what fonts and colors they're using?

You would use Chrome DevTools.

Steps:

Right-click on the element (like a heading).

Click Inspect.

In the Elements panel:

Look at the Styles tab to see the applied CSS rules.

Look at the Computed tab to see the final font-family and color values.

You can also check the linked CSS files in the Sources tab.

DevTools shows:
• The exact font-family
• The color (hex, rgb, etc.)
• Which CSS rule applied it
• Whether something was overridden

Short answer version:

I would use browser DevTools to inspect the element and check the applied styles and computed values.

# Why do we recommend external CSS files over inline styles, even though inline styles win in specificity?

This is testing architectural thinking.

Even though inline styles override others, we avoid them because:

1️⃣ Separation of concerns

HTML = structure
CSS = styling

Mixing them makes code harder to manage.

2️⃣ Maintainability

Changing inline styles requires editing every element.
External CSS allows one change to affect the whole site.

3️⃣ Reusability

Classes and external styles can be reused across pages.

4️⃣ Performance

External CSS files are cached by the browser.
Inline styles must be loaded repeatedly.

5️⃣ Cleaner code

External CSS keeps HTML readable and professional.

# if your friend's <img> wont center even if margin: 0 auto
The image is still inline (most common issue)

margin: 0 auto; only works on block-level elements.

By default, img is inline, so it won’t center.
