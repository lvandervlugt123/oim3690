# api cont.
fetch() — Getting Data from an API
fetch() is built into every browser. Every time you use it, you follow the same 3 steps: fetch the URL, convert the response to JSON, then use the data.

You need await to tell JavaScript to pause and wait for the data to actually arrive before moving on.

await can only be used inside a function marked as async. You put async before the function definition, and then use await inside it whenever you're waiting on data.

APIs can fail due to network issues, bad URLs, or server errors. You should always wrap your fetch call in a try/catch block — the try part runs your normal code, and the catch part handles anything that goes wrong, like showing an error message to the user.


# Get vs post

- There are two main ways to interact with an API. GET means you're asking for data like looking something up. 
- POST means you're sending data ,like submitting a form. GET is the default; POST requires extra options like a method, headers, and a body.

- Once you have data from an API, you display it using the same DOM and loop techniques from your JavaScript basics. You fetch the data, loop through it, create HTML elements for each item, and append them to the page.

# Protecting API Keys

Never put an API key directly in your main code file if it's going to GitHub
Store it in a separate file (like config.js) and add that file to .gitignore so Git never tracks it
Load the config file in your HTML before your main script
Even with this setup, keys are still visible in the browser's DevTools — for class projects that's fine, but real apps need a backend to keep keys truly secret



# images
What is an Image?

Every digital image is a grid of tiny colored squares called pixels. For example, a 1920x1080 image has over 2 million pixels. Each pixel stores a color value.

Each pixel stores 4 values, known as RGBA:

R (Red) — intensity of red, from 0 to 255
G (Green) — intensity of green, from 0 to 255
B (Blue) — intensity of blue, from 0 to 255
A (Alpha) — transparency, from 0 (invisible) to 255 (fully visible)

# image data as an array
Pixel data is stored as one long flat array. Every pixel takes up 4 spots in the array — one for each RGBA value. So a 2x2 image (4 pixels) would have 16 values in the array. The pattern repeats: R, G, B, A, R, G, B, A, and so on.
